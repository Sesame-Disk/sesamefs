  [FIXED] Files opened from search return 404/500 (2026-02-12)
  - ROOT CAUSE 1 (404 on .docx): OnlyOffice getFileID() queried `libraries` table with
    `WHERE org_id = ? AND library_id = ?` — if org_id didn't match the library's partition
    key, the query failed silently → 404 "File Not Found" error page.
    Fix: Changed to `libraries_by_id WHERE library_id = ?` (no org_id dependency),
    consistent with FSHelper.GetRootFSID().
  - ROOT CAUSE 2 (500 on .pdf): serveInlinePreview() extracted the auth token from
    the query param or Authorization header, but when the user arrived without a token
    (anonymous/dev mode), it generated `?token=` (empty) in the raw embed URL →
    the sub-request to /repo/:id/raw/* failed with 500.
    Fix: Improved token extraction (supports Token/Bearer), added fallback to dev
    token when in dev mode.
  - ROOT CAUSE 3 (no token in URL): All 6 onSearchedClick() handlers across the
    frontend opened files in new tabs via window.open() without including the auth
    token in the URL. New tabs can't use localStorage/headers, so they arrived
    unauthenticated.
    Fix: All handlers now call getToken() and append ?token= to the URL.
  Files changed: internal/api/v2/fileview.go, internal/api/v2/onlyoffice.go,
    frontend/src/app.js, frontend/src/settings.js, frontend/src/repo-history.js,
    frontend/src/repo-snapshot.js, frontend/src/repo-folder-trash.js,
