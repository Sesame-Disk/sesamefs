1. SECURITY (Open)
=======================================

1.1 CORS Defaults to Allow All Origins [MEDIUM]
    File: internal/api/server.go (~line 175)
    - In production, if CORS.AllowedOrigins is not configured, defaults to AllowAllOrigins = true
    - Should default to same-origin or reject if unconfigured
    - Dev mode allowing all origins is fine

1.2 Silent DB Error Swallowing in Sync [LOW]
    File: internal/api/sync.go
    - Block metadata write failures silently fall back to "hot" storage class
    - Errors should at minimum be logged
    - Not a data loss risk, but makes debugging harder

1.3 Anonymous Auth Bypass in Dev Mode [LOW - dev only]
    File: internal/api/server.go (~line 870)
    - When AllowAnonymous && DevMode, requests without auth token get
      authenticated as the first dev token user
    - Only affects dev mode, not production — low risk
