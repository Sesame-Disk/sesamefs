# Frontend Environment Configuration

This directory contains frontend-specific environment variables for the SesameFS web UI.

## Configuration Files

- **`.env`** - Your local configuration (not committed to git)
- **`.env.example`** - Template with all available options (committed to git)

## Quick Start

```bash
# Copy the example file to create your local config
cp .env.example .env

# Edit .env to customize for your environment
nano .env
```

## Environment Variables

### API Configuration

- **`SESAMEFS_API_URL`** - Backend API URL
  - Leave empty for same-origin (when frontend is served by SesameFS)
  - Set to full URL for separate deployment (e.g., `http://localhost:8080`)

### Build Optimization

These settings control Docker build memory usage and performance:

- **`WEBPACK_PARALLEL_BUILD`** - Enable/disable parallel webpack processing
  - `true` - Faster builds, uses more memory (good for local development)
  - `false` - Slower builds, uses less memory (required for Docker with limited RAM)
  - Default: `false`

- **`GENERATE_SOURCEMAP`** - Enable/disable source map generation
  - `true` - Generate source maps (useful for debugging)
  - `false` - Skip source maps (saves memory during build)
  - Default: `false`

- **`NODE_MAX_MEMORY`** - Node.js memory limit in MB
  - Controls `--max_old_space_size` Node option
  - Default: `4096` (4GB)
  - Increase if build fails with OOM
  - Decrease to save memory

## Docker Build

The Dockerfile reads these settings automatically:

```bash
# Build with default settings from .env
docker-compose build frontend

# Override specific settings
docker-compose build frontend --build-arg SESAMEFS_API_URL=http://example.com
```

## Local Development

For local development (not Docker):

```bash
# Development server (hot reload)
npm start

# Production build
npm run build
```

The build will respect settings in `.env`.

## Troubleshooting

### Build fails with "Killed" or "cannot allocate memory"

1. Set `WEBPACK_PARALLEL_BUILD=false` in `.env`
2. Set `GENERATE_SOURCEMAP=false` in `.env`
3. Increase `NODE_MAX_MEMORY` (try 6144 or 8192)
4. Remove Elasticsearch from docker-compose if running

### Build is too slow

1. Set `WEBPACK_PARALLEL_BUILD=true` in `.env`
2. Build locally instead of in Docker: `npm run build`
3. Use existing frontend image: `docker-compose up -d` (skips rebuild)

## Separation from Backend Config

The **root `/.env`** file is for backend services (Cassandra, S3, etc.).

The **`frontend/.env`** file is for frontend build and runtime settings.

Keep them separate to avoid confusion and namespace conflicts.
